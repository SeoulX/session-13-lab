Which flag makes claude run non-interactively, exiting after one prompt?
Shorthand for --print. Runs one prompt, prints result to stdout, exits. The other options do not exist.
Q2
· 1 pt
What exit code means the CLI call ran successfully?
POSIX convention. Any non-zero code = failure. Lets if claude -p "..."; then ... work in shell scripts and CI stages.
💀 Moral: kung galit ang shell, may exit code na 1. Kung tahimik, 0. Simple.
Q3
· 3 pts
Which two output formats does --output-format accept for programmatic use?
json = single envelope after the run (best for one-shot). stream-json = one event per line as Claude works (best for live monitoring + long agent runs).
Q4
· 1 pt
In --output-format json, which field carries the answer text?
Envelope also carries .sessionId, .total_cost_usd, .numTurns. Use jq -r '.result' to fish out just the answer.
Q5
· 1 pt
Which extra flag does --output-format stream-json require in CLI 2.1+?
CLI 2.1.215 requires --verbose whenever stream-json is used. Without it the command errors and exits non-zero — silent breakage in scripts written for older CLI versions.

Name two things that DO carry into a claude -p call from your interactive environment.
~/.claude settings + project CLAUDE.md. Also allowed-tools list, MCP server config. Does NOT carry: prior conversation, plan-mode state, session todo list. Every -p call is fresh.

In GitHub Actions, what makes two jobs: run in parallel?
GHA runs jobs concurrently by default. needs: [job-x] serializes them. Parallelism is the default; sequencing is the exception.

Wall clock = max(15, 8, 20) = 20. Sum would be 43. Claude joins CI at ZERO added minutes if your slowest existing job is already slower than Claude.

What is the correct JSON field name for cost in a stream-json result event?
Common bug: code written as .totalCost (camelCase) reads null → sums to $0.0000. Real field is snake_case.
💸 "Bakit $0 lahat ng runs?" — spoiler: tinatawagan mo yung field na wala.
Q15
· 5 pts
Where do you find tool_use events inside a stream-json line?
tool_use events nest INSIDE assistant messages. Correct jq: jq -r '.message.content[]? | select(.type=="tool_use") | .name'. Top-level .type is "assistant", "user", "result".
Pulse · applied arc
Q19
· 3 pts
Pulse's tag lifecycle now cuts a NEW tag on every submit instead of destructively re-cutting v0.0.0. Name one reason this matters.
Preserves Bitbucket tag history + Jenkins build lineage. Old behavior deleted v0.0.0 and re-pushed → history gone, Jenkins build numbers collided. New behavior: every attempt keeps its own tracker URL forever.
📼 Marites: "Sino nag-delete ng tag ko?" — before: Pulse. After: walang gumagalaw.

In the 3-tier spec lookup (/api/deployments/spec/{slug}?tag=X), which tier fires FIRST?
